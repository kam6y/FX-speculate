## 1. システム概要

### 1.1 プロジェクト目的

Temporal Fusion Transformer（TFT）をベースとしたUSD/JPY為替レート予測システムを構築する。日米金利差・マクロ経済指標・テクニカル指標を統合し、多段階予測（1日〜4週間先）および不確実性推定（分位点予測）を提供する。

### 1.2 スコープ

- **予測対象:** USD/JPY日次・週次終値
- **予測ホライズン:** 1日・5日・10日・20日先
- **出力:** 点予測 + 10%/50%/90%分位点予測
- **目標方向精度:** 58%以上（ランダムウォーク比）
- **ベンチマーク:** ARIMA・LSTM・XGBoostとの比較評価

### 1.3 システム構成図

## 2. TFTアーキテクチャ詳細

### 2.1 Temporal Fusion Transformerの概要

TFTはGoogle Brainが2021年に発表した時系列予測モデルであり、以下の4つの主要コンポーネントで構成される。

| コンポーネント | 役割 | FX予測での活用 |
|:---|:---|:---|
| **Variable Selection Network (VSN)** | 入力変数の重要度を自動学習し、不要な変数を抑制 | 金利差・VIX・テクニカル指標の重要度を自動判定 |
| **Gated Residual Network (GRN)** | 非線形処理とゲート機構による効率的な情報フロー制御 | 市場レジームに応じた動的な特徴量重み付け |
| **Temporal Self-Attention** | 過去のどの時点が予測に寄与するかを学習 | FOMC・日銀会合前後の重要時点を自動検出 |
| **Quantile Output Layer** | 10%/50%/90%分位点で予測し不確実性を推定 | リスク管理・ポジションサイズ決定に活用 |

### 2.2 ハイパーパラメータ設計

| パラメータ | 推奨値 | 備考 |
|:---|:---|:---|
| hidden_size | 160 | VSN・GRNの隠れ層次元数 |
| attention_head_count | 4 | マルチヘッドアテンションのヘッド数 |
| dropout | 0.1 〜 0.3 | 過学習防止。データ量に応じて調整 |
| hidden_continuous_size | 8 | 連続変数の埋め込み次元 |
| learning_rate | 1e-3 〜 3e-3 | OneCycleLRでスケジューリング |
| max_encoder_length | 60（日次） | 過去60日分のエンコーダ入力 |
| max_prediction_length | 20（日次） | 20日先までの同時予測 |
| batch_size | 64 | GPUメモリに応じて調整 |
| max_epochs | 100 | EarlyStopping (patience=10) 併用 |
| quantiles | [0.1, 0.5, 0.9] | 分位点損失関数に使用 |

## 3. データパイプライン設計

### 3.1 入力特徴量一覧

TFTは入力変数を以下の3カテゴリに分類して扱う。この分類が他のモデルとの最大の差別化ポイントである。

#### 3.1.1 既知未来変数（time_varying_known）

予測時点で将来の値が確定している変数。カレンダー情報や予定されたイベントが該当する。

| 特徴量 | 説明 | データソース |
|:---|:---|:---|
| 曜日 (day_of_week) | 0（月）〜4（金）のカテゴリ変数 | 自動生成 |
| 月 (month) | 1〜12の周期変数 | 自動生成 |
| 四半期 (quarter) | Q1〜Q4の周期変数 | 自動生成 |
| FOMC会合日 | FOMC開催日からの距離（日数） | FRBカレンダー |
| 日銀金融政策決定会合日 | 会合日からの距離（日数） | 日銀カレンダー |
| 米国雇用統計発表日 | 発表日からの距離（日数） | BLSカレンダー |
| 祭日フラグ | 日米各市場の休場日フラグ | 各国祖日カレンダー |

#### 3.1.2 時変共変量（time_varying_unknown）

過去の値のみ利用可能で、将来の値が不明な変数。市場データおよびテクニカル指標が該当する。

| 特徴量 | 説明 | データソース |
|:---|:---|:---|
| 終値 (close) | USD/JPY日次終値 | Yahoo Finance |
| リターン (log_return) | 対数リターン: ln(Ct/Ct-1) | 算出値 |
| RSI(14) | Relative Strength Index（14日） | ta-lib |
| MACD | MACDライン + シグナルライン | ta-lib |
| ボリンジャーバンド | 20日MA±2σの位置 | ta-lib |
| ATR(14) | Average True Range（14日） | ta-lib |
| 日米10年債利回り差 | US10Y - JP10Yスプレッド | FRED API |
| VIX | CBOEボラティリティ指数 | Yahoo Finance |
| 金価格 (XAU/USD) | リスクオフ指標 | Yahoo Finance |
| ニュースセンチメント | NLPベースの市場センチメントスコア | FinBERT / NewsAPI |

#### 3.1.3 静的共変量（static）

時間によって変化しないメタ情報。単一通貨ペアの場合は最小限だが、マルチペア拡張時に重要となる。

- **currency_pair_id:** 通貨ペア識別子（マルチペア拡張用）
- **market_regime:** 市場レジームラベル（トレンド/レンジ/高ボラティリティ）

### 3.2 データ前処理フロー

1. **欠損値処理:** 祝日は前営業日の値で前方増補。マクロ指標は線形補間。
2. **正規化:** 各特徴量をローリングウィンドウで標準化（Z-score）。ウィンドウ幅はエンコーダ長と同一の60日。
3. **外れ値処理:** リターンの±4σ超をクリッピング。急激な主要イベント時のデータは保持。
4. **データ分割:** 時系列分割。訓練 70% / 検証 15% / テスト 15%。Walk-forward方式で拡張ウィンドウ検証も実施。

## 4. 学習パイプライン

### 4.1 学習フロー

| フェーズ | 内容 | 詳細 |
|:---|:---|:---|
| **Phase 1** | データ収集・前処理 | 過去5年分の日次データを収集。特徴量エンジニアリング実施。 |
| **Phase 2** | ベースライン学習 | ARIMA・LSTM・XGBoostのベンチマークモデルを学習。 |
| **Phase 3** | TFTモデル学習 | PyTorch ForecastingでTFTを学習。Optunaでハイパラチューニング。 |
| **Phase 4** | アンサンブル構築 | TFT + XGBoostのスタッキングアンサンブル。メタモデルはRidge回帰。 |
| **Phase 5** | バックテスト・評価 | ウォークフォワードバックテストで実運用性能を検証。 |

### 4.2 損失関数

**Quantile Loss（分位点損失）:** 各分位点qに対して、過小評価と過大評価に非対称なペナルティを与える。q=0.1では実績が予測を下回るケースに大きなペナルティ、q=0.9ではその逆となる。

```
L_q(y, ŷ) = max[q(y - ŷ), (q - 1)(y - ŷ)]
```

最終損失は全分位点・全ホライズンの平均とする。

### 4.3 最適化戦略

1. **オプティマイザ:** AdamW（weight_decay=1e-2）
2. **スケジューラ:** OneCycleLR（max_lr=3e-3, pct_start=0.3）
3. **早期停止:** patience=10, monitor=val_loss, mode=min
4. **ハイパラチューニング:** Optuna TPESampler（n_trials=50）でhidden_size, dropout, learning_rateを探索

## 5. 評価指標・バックテスト設計

### 5.1 評価指標

| 指標 | 説明 | 目標値 |
|:---|:---|:---|
| MAE | 平均絶対誤差。予測値と実績の平均差 | < 0.5円（1日先） |
| RMSE | 平均二乗誤差平方根。大きな誤差に敏感 | < 0.7円（1日先） |
| MAPE | 平均絶対パーセント誤差 | < 0.5% |
| 方向精度 | 上昇/下降方向の正答率 | > 58%（1日先） |
| Quantile Loss | 分位点予測のキャリブレーション精度 | カバレッジ率が分位点に近似 |
| Sharpe Ratio | リスク調整後リターン | > 1.0（バックテスト） |
| Max Drawdown | 最大ドローダウン | < 15% |

### 5.2 バックテスト方法

**Walk-Forward Validation:** 実運用環境を再現するため、学習ウィンドウを段階的に拡張しながら評価する。

- **初期学習ウィンドウ:** 3年分（約750営業日）
- **テストウィンドウ:** 20営業日（1ヶ月）ごとにスライド
- **再学習:** 毎回ウィンドウを拡張してfine-tuning（20エポック）
- **取引シミュレーション:** スプレッド・スリッページ・手数料を考慮した現実的な収益計算

## 7. 運用・デプロイ設計

### 7.1 推論パイプライン

1. **毎日 17:00 JST:** 当日の市場データ収集・特徴量更新
2. **毎日 17:30 JST:** TFT推論実行、予測結果をDBに保存
3. **毎日 18:00 JST:** ダッシュボード更新・アラート配信

### 7.2 モデル再学習ポリシー

- **定期再学習:** 毎日全体再学習。
- **ロールバック:** 新モデルが旧モデルより明確に悪い場合、自動的に前バージョンに戻す。

### 7.3 アラート・モニタリング

| アラート種別 | 条件 | 通知先 | アクション |
|:---|:---|:---|:---|
| 精度低下 | 方向精度 < 50%（10日） | ダッシュボード | 自動再学習開始 |
| データ欠損 | APIエラー継続 | ダッシュボード | フォールバックデータソース切替 |
| 異常予測 | 予測値が±5円超 | ダッシュボード | 手動確認・予測無効化 |
| ドリフト検知 | PSI > 0.2 | ダッシュボード | 全体再学習検討 |

## 8. リスクと制約

### 8.1 モデルリスク

| リスク | 説明 | 緩和策 |
|:---|:---|:---|
| 過学習 | TFTはパラメータ数が多く、データ不足時に過学習リスクが高い | Dropout + Early Stopping + Regularization |
| レジーム変化 | 市場構造の変化によりモデル性能が急激に劣化 | ドリフト検知 + 自動再学習 |
| データ品質 | API障害・データソース変更による入力欠損 | 複数データソース + 整合性チェック |
| ブラックスワン | 予測不能な極端な市場イベント | ポジションサイズ制限 + ストップロス |

### 8.2 重要な注意事項

本システムは実験的な予測ツールであり、実際の投資判断に使用する場合は十分なリスク管理が必要である。過去のパフォーマンスは将来の結果を保証するものではない。特にFX市場は効率的市場仮説に近いとされ、一貫した超過リターンの獲得は極めて困難である。
