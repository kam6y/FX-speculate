# 3モデル比較レポート

**実施日**: 2026-03-20
**目的**: USD/JPY 1分足方向予測における3モデル（CatBoost / LightGBM / Transformer）の性能比較
**手法**: 各モデルに対しRalph Loop 15イテレーションのヒルクライム最適化を実施

## モデル別分析

### Transformer（1位）— スナイパー型

- **強み**: 極めて高い選択性。51件中59%で勝ち、avg_win/avg_loss比 = 1.38
- **弱み**: 取引数が少なすぎ（51件）、統計的信頼性に懸念
- **特性**: 高確率閾値（0.40-0.80）で低確信度予測を排除。確率分布が0.35-0.40付近に集中しており、閾値を0.05下げるだけで取引数が640倍に暴走
- **最適構成**: 小さいモデル（d_model=32, 1層）、window_size=100、全99特徴量使用
- **Ralph Loop効果**: イテレーション2で確率閾値を引き上げただけで Sharpe -7.96 → +3.30 に劇的改善

### CatBoost（2位）— バランス型

- **強み**: 1,268件の取引で統計的に十分なサンプル数。avg_win ≈ avg_loss でバランスが取れている
- **弱み**: Sharpe +0.229は非常に薄いエッジ。Optuna目的関数自体がマイナス（訓練データでも利益が出ない）
- **特性**: seed変更（42→123）が最も効果的だった＝安定した良解がまばらにしか存在しない
- **最適構成**: Top40特徴量、時間特徴量4個上限、PIPS閾値5-15、RANDOM_SEED=123
- **Ralph Loop効果**: 15回中3回のみ改善（20%）。大半はロールバック

### LightGBM（3位）— 改善困難型

- **強み**: 正則化強化（lr制限、depth制限、reg引き上げ）で過学習を抑制する方向で改善
- **弱み**: ベストでもSharpe -0.968。コスト後に利益を出せない
- **特性**: CatBoostと同一パイプラインだが、LightGBMの方がハイパーパラメータ感度が高い
- **最適構成**: CV内n_estimators=500, early_stopping=20で過学習抑制
- **Ralph Loop効果**: 15回中3回のみ改善（20%）。CatBoostと同じ改善率

## 3モデル共通の知見

### 効果があった変更（全モデル共通）

1. **確率閾値の引き上げ**: 3モデルすべてで最大の改善要因。低確信度トレードの排除が最重要
2. **過学習抑制**: 正則化強化、モデル複雑度の削減がプラスに寄与

### 効果がなかった変更（全モデル共通）

1. **特徴量数の増加**: 過学習を招く（CatBoost It4, LightGBM It3）
2. **Optunaトライアル数の増加**: 必ずしも改善せず（CatBoost It7/11, LightGBM It5）
3. **閾値グリッドの細分化**: バリデーション過適合（CatBoost It6, LightGBM It8）
4. **時間特徴量の除外**: 予測能力が低下（CatBoost It8, LightGBM It7）

### 根本的な課題

- **1分足FXの信号対雑音比が極めて低い**: 取引コスト（スプレッド0.2pips + スリッページ0.1pips + API手数料0.002%）を超えるエッジの確保が困難
- **Sharpe vs 取引数のトレードオフ**: 高確率閾値でSharpeを上げると取引数が激減。取引数を増やすとSharpeが崩壊。このトレードオフは3モデル共通
- **Optuna探索のランダム性依存**: seed変更が構造的な改善より効果的だったケースが複数（CatBoost It10, Transformer It7）

## 推奨事項

### 短期（リアルタイム推論に向けて）

設計書の方針通り、**Transformer**を最良モデルとして選定し、リアルタイム推論（GMO Coin FX API接続）を実装する。ただし以下の注意点がある:

- 取引数が少ないため、十分な期間のフォワードテストが必要
- 1日あたり0.3件程度のシグナルしか出ない計算（51件 / テスト期間）
- Max Drawdown -2,319円は小さいが、サンプル数が少ないため信頼区間が広い

### 中期（モデル改善）

| 優先度 | 施策 | 期待効果 |
|--------|------|----------|
| 高 | データ量の増加（より長期間のデータ取得） | 全モデルの汎化性能向上 |
| 高 | マルチタイムフレーム特徴量（5分/15分/1時間足） | 信号対雑音比の改善 |
| 中 | 予測ホライゾンの拡大（15分→30分/60分） | より強い方向性シグナル |
| 中 | アンサンブル（複数seed/モデルの予測平均） | Transformer It14で+0.74, 71件 → 有望 |
| 低 | ボラティリティ調整済みターゲット | ラベルの安定性向上 |

### Ralph Loop プロセスの評価

- **改善率 20%**: 3モデルとも15回中3回のみ改善。FX 1分足の最適化は「砂漠でオアシスを探す」ような探索
- **papermill**: nbconvertのカーネル死亡問題をpapermillで解決。GPU + Windows環境では必須
- **ヒルクライム方式の有効性**: ベスト保持 + ロールバック機構により、悪い変更が蓄積されない。安全な探索ができた
