## プロジェクト全体構造

```
FX-speculate/
├── Temporal Fusion Transformer/   ← メインの予測システム
│   ├── train_tft.py               ← 学習パイプライン
│   ├── evaluate_tft.py            ← 評価・バックテスト

│   └── dashboard/                 ← Webダッシュボード

│       ├── database.py            ← SQLiteデータベース管理
│       ├── predictor.py           ← 推論パイプライン
│       ├── templates/             ← HTMLテンプレート
│       ├── static/                ← CSS/JS
│       └── model/                 ← デプロイ済みモデル
│
├── ralph-loop/                    ← 自律改善エージェント（1分足モデル）
├── data/                          ← 生データ（Parquet形式）
├── scripts/                       ← 共有ユーティリティ
├── notebooks/                     ← レガシーモデル（CatBoost等）
├── docs/                          ← ドキュメント

```

## ファイル一覧と役割

### 学習系
| ファイル | 役割 | 比喩 |
|---------|------|------|
| `train_tft.py` | データ取得→特徴量作成→モデル学習→保存 | 「料理教室の先生＋厨房」：材料仕入れから調理まで一貫 |
| `evaluate_tft.py` | 保存済みモデルの性能を多角的に評価 | 「食品検査場」：できた料理の品質チェック |

### ダッシュボード系
| ファイル | 役割 | 比喩 |
|---------|------|------|
| `dashboard/app.py` | FastAPIサーバー＋日次予測スケジューラ | 「店舗のフロント」：注文受付＋定時オペレーション |
| `dashboard/predictor.py` | 最新データで推論を実行 | 「店舗の厨房」：注文が来たら料理する |
| `dashboard/database.py` | 予測結果・指標・アラートをSQLiteで管理 | 「在庫管理帳簿」：何をいつ作ったか記録 |

### 共有ライブラリ
| ファイル | 役割 |
|---------|------|
| `scripts/data_loader.py` | 生データの読み込み |
| `scripts/features.py` | 特徴量エンジニアリング |
| `scripts/evaluation.py` | 評価メトリクス計算 |

## データフロー

### 学習時
```
Yahoo Finance / FRED API
       │
       ▼
  生の市場データ（日足 OHLCV、金利、VIX等）
       │
       ▼
  特徴量エンジニアリング
  ├─ テクニカル指標（RSI, MACD, ボリンジャーバンド, ATR）
  ├─ カレンダー特徴（曜日, 月, 四半期）
  ├─ イベント距離（FOMC/日銀会合まで何日）
  ├─ 祝日フラグ（米国/日本）
  └─ マーケットレジーム（トレンド/レンジ/高ボラ）
       │
       ▼
  前処理（Z-score正規化、外れ値クリッピング）
       │
       ▼
  TimeSeriesDataSet（PyTorch Forecastingのデータ入れ物）
  ├─ 過去120日 → エンコーダ入力
  └─ 未来10日  → デコーダ（予測対象）
       │
       ▼
  TFTモデル学習（PyTorch Lightning）
       │
       ▼
  チェックポイント保存 → artifacts/checkpoints/
```

### 推論時（毎日17:30 JST）
```
APScheduler起動
       │
       ▼
  最新の市場データ取得（学習時と同じパイプライン）
       │
       ▼
  保存済みモデルで予測（10日先まで × 3分位点）
       │
       ▼
  SQLiteに保存 → ダッシュボードに表示
       │
       ▼
  アラートチェック（精度低下、異常値等）
```

## 技術スタック

| 領域 | 使用技術 | 初心者向け説明 |
|------|---------|-------------|
| データ取得 | yfinance, FRED API | 株価や経済指標をネットから自動取得 |
| 特徴量計算 | ta-lib, pandas | テクニカル指標の計算 |
| モデル | PyTorch + PyTorch Forecasting | ニューラルネットワークの枠組み＋時系列専用ツール |
| 学習管理 | PyTorch Lightning | 学習ループを自動化するヘルパー |
| ハイパラ探索 | Optuna | 最適な設定値を自動で見つける |
| Webサーバー | FastAPI + Uvicorn | APIサーバー（Flaskの高速版） |
| 定時実行 | APScheduler | Pythonのcronジョブ |
| データベース | SQLite | ファイル1つで動く軽量DB |
| パッケージ管理 | UV | pip/poetryの高速版 |

## TFTモデルの入力特徴量

### 時系列変動（過去のみ利用可能）
終値、ログリターン、RSI(14)、MACD、ボリンジャーバンド、ATR(14)、
米日金利差、VIX、金価格、ニュースセンチメント

### 時系列変動（未来も既知）
曜日、月、四半期、FOMC/日銀会合までの日数、祝日フラグ

### 静的（不変）
通貨ペアID、マーケットレジームラベル

## 主要な設定値

| 設定 | 値 | 意味 |
|------|---|------|
| MAX_ENCODER_LENGTH | 120日 | モデルが参照する過去データの長さ |
| MAX_PREDICTION_LENGTH | 10日 | 何日先まで予測するか |
| QUANTILES | [0.1, 0.5, 0.9] | 悲観・中央・楽観の3パターン |
| HIDDEN_SIZE | 192 | モデル内部の情報処理の幅 |
| BATCH_SIZE | 64 | 一度に処理するデータ数 |
| MAX_EPOCHS | 50 | 最大学習回数 |
| PATIENCE | 8 | 改善しなくなってから待つ回数 |
